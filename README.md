## Environment Setup
- Run `uv pip install -e ".[test]"` to install the Triton model client and dependencies.  This should be installed in the same venv as the Globus Compute Endpoint serving the Triton models.
- `python register_globus_compute_func.py` -- registers the GC client function that invokes the local triton server.
- `user_config_template.yaml.j2` -- Globus Compute Endpoint configuration for Sophia.  Starts the models following the same pattern as `serve-models.sh`.  This is copied into your actual GC endpoint.

## Triton Client
- `triton_hep_client.py` -- implements the `TritonHEPClient` class that wraps each gRPC inference request

## Testing the Triton Client on a Sophia node
- `serve-models.sh` -- run this on a Sophia compute node to spin up the models for testing
- `pytest test_client.py` -- on the same node as the running models: tests each of the model types through the `TritonHEPClient` class.

## Sample data for end-to-end testing:
- `sample_inputs` contains sample input files for each model type (generated by `generate_sample_inputs.py`)
